# 个人支出管理 App 设计文档

## 概述

一个 Android 平台个人支出管理应用，支持手动记账、截图自动识别、报销管理、数据报表和数据导出/恢复。纯本地运行，无需联网。

## 技术方案

- **框架**：Flutter 3.x + Dart
- **本地数据库**：sqflite（SQLite）
- **OCR 识别**：google_mlkit_text_recognition（离线识别）
- **图表展示**：fl_chart
- **CSV 导出/导入**：csv 库

## 页面结构

底部导航栏 4 个标签页 + 弹出式记账表单：

| 标签 | 功能 |
|------|------|
| 首页 | 月度概览卡片 + 最近支出列表 + "记一笔"按钮 |
| 记账 | 全部流水列表，可按日期/类别筛选，每条显示金额/类别/备注/报销状态/截图标记 |
| 报表 | 月份切换 + 总支出卡片 + 分类支出排行 + 支付方式占比 + 每日趋势 |
| 设置 | 分类管理大表 + 数据导出 + 数据导入 |

## 数据模型

### records（支出记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| amount | REAL | 金额 |
| category_id | INTEGER | → categories.id |
| note | TEXT | 备注 |
| payment | TEXT | 支付方式（微信/支付宝/现金/银行卡） |
| datetime | TEXT | 完整时间（yyyy-MM-dd HH:mm:ss） |
| created_at | TEXT | 记录创建时间 |

### categories（分类）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| name | TEXT | 分类名称 |
| icon | TEXT | 图标 emoji |
| parent_id | INTEGER | null=一级分类，非空=子类别 |
| sort_order | INTEGER | 排序 |

### reimbursements（报销）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| record_id | INTEGER | → records.id |
| status | TEXT | pending / done |
| reimbursed_date | TEXT | 核销时间 |
| note | TEXT | 报销备注 |

### attachments（截图附件）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| record_id | INTEGER | → records.id |
| file_path | TEXT | 本地文件路径 |
| created_at | TEXT | 保存时间 |

## 核心功能

### 手动记账

点击"＋记一笔"弹出表单：
- 金额输入（大号数字）
- 类别选择（一级 → 子类别）
- 支付方式选择
- 时间（默认当前时间）
- 备注
- 添加截图（可选）
- 标记待报销（开关）

### 截图识别

两种触发方式：
1. **自动监听**：在 App 内截图 → 弹窗询问是否识别
2. **从相册选**：记账时点"添加截图" → 从相册选取

OCR 提取信息：
- 金额（识别"实付/合计 ¥XX"）
- 时间（提取支付时间）
- 商户名 → 填入备注
- 类别（关键词匹配，如"午餐"→ 餐饮）
- 支付方式（识别"微信支付"等）

识别到的字段自动填入，未识别到的留空，用户确认后保存。

### 报销管理

- 记账时开关"待报销"
- 报销列表查看所有垫付款
- 收到报销款后核销（标记 done + 核销日期）

### 报表

- 月份切换（左右箭头）
- 本月汇总：总支出、笔数、日均、待报销
- 分类支出排行（柱状图 + 金额）
- 支付方式占比
- 每日支出趋势图

### 数据导出/导入

- **导出**：CSV 文件，可选"包含截图"（勾选时导出为 ZIP：CSV + 图片文件夹）
- **导入**：选择之前导出的 ZIP 文件，恢复所有数据及截图

### 分类管理（设置页）

统一表格展示所有类别（一级 + 子类别），支持：
- 新增（顶部"＋"按钮）
- 编辑分类名称/图标
- 删除分类
